# Лабораторна робота 6

## Тема
Хешування, автентифікація повідомлень і виведення криптографічних ключів.

## Мета
Ознайомитися з практичним використанням Argon2 для зберігання паролів, HMAC-SHA256 для контролю цілісності та автентичності повідомлень, а також PBKDF2-HMAC-SHA256 для виведення криптографічного ключа з пароля.

## Завдання
1. Реалізувати хешування та перевірку паролів користувачів за допомогою Argon2.
2. Згенерувати HMAC-SHA256 для повідомлення та показати провал перевірки після зміни повідомлення.
3. Вивести 256-бітний ключ із пароля за допомогою PBKDF2-HMAC-SHA256.

## Короткі пояснення
- **Hash**: одностороннє перетворення даних у фіксований рядок байтів.
- **MAC**: код автентифікації повідомлення, який підтверджує цілісність і знання спільного секретного ключа.
- **KDF**: функція виведення ключа, що перетворює пароль або секрет у криптографічний ключ потрібної довжини.
- **Salt**: випадкове значення, яке додається до пароля перед обробкою, щоб однакові паролі не давали однаковий результат.
- **Iterations**: кількість повторень алгоритму; більше ітерацій підвищує вартість перебору.

## Чому не можна зберігати паролі як звичайний SHA-256
SHA-256 є швидкою хеш-функцією, тому вона погано підходить для паролів: зловмисник може дуже швидко перебирати варіанти паролів на CPU, GPU або спеціалізованому обладнанні. Для паролів слід використовувати спеціальні алгоритми на кшталт Argon2, які є повільнішими та стійкішими до масового перебору.

## Чому для автентифікації використовують HMAC, а не звичайний хеш
Звичайний хеш повідомлення не доводить, хто саме його сформував, бо будь-хто може обчислити той самий хеш. HMAC використовує секретний ключ, тому лише сторони, що знають ключ, можуть створити коректний MAC і перевірити справжність повідомлення.
